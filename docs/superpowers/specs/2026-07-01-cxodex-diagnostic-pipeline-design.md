# CXODEX 网站重构：诊断流水线 + 首页麦肯锡化 — 设计 spec

日期：2026-07-01
提出人：用户（CEO，业主）
核心：把 cxodex.com 从「工具集合」变成 **一个入口 + 一个判断 + 一个动作**——输入企业数据 → 5 分钟出《CXO 企业体检报告》→ 报告把根因唯一化并路由到三大服务。

## 锁定决策（用户已定）
1. 诊断向导 = **独立香港子域 `diagnose.cxodex.com`**（HK VPS 自托管，国内快、自包含；架构同 oee-web：单页 + server.js + PM2 + nginx + certbot）。
2. 现有 `diagnosis.cxodex.com`（38 题问卷）**降级**为"定性深挖/备选"，新向导成为主诊断入口；旧问卷不删（可回滚）。
3. **分阶段：先首页（阶段1），再向导（阶段2）**。

## 三大服务（贯穿全站的定位主线）
1. **帮企业扭亏**：精益（Lean）治浪费与生产瓶颈 + 6 Sigma 治质量波动。
2. **产品战略与国际业务**：有效打法让企业短期获得大量订单。
3. **组织人事变革管理**。

## 阶段 1：首页重构（本次先做）

主站 www.cxodex.com（Next.js，Vercel）。目标：新定位立住 + 麦肯锡式质感 + 突出三大重点与客户，弱化重复个人介绍，保留 5 工具入口。

### 1.1 设计语言（麦肯锡式）
- **版式纪律**：大留白、清晰栅格、发丝分隔线、内容前置、全幅图。
- **字体**：编辑级衬线做大标题（`.serif`，现有系统衬线栈）、克制无衬线做正文。标题字号加大、字重加强。
- **配色**：保留 **CXODEX 金**（#b0832f，取自 logo）为唯一品牌强调色 + 深墨 + 暖纸底（麦肯锡式克制单强调）。（可选：若用户要麦肯锡深藏青，改 `--brand` 即可，默认金。）
- **LOGO**：首屏左上金色 C 标 + CXODEX 字标**显著放大、加对比**（当前偏小）。页眉/页脚一致。

### 1.2 首页结构（自上而下）
1. **页眉**：放大的 LOGO + 极简导航 +「开始企业诊断」金色主按钮 + 返回/预约。
2. **首屏 Hero（一句话定位，讲入口不讲能力）**：
   - 主标题（衬线大字）：*输入企业数据，5 分钟生成生产与利润诊断报告*
   - 副标题一行：以 30 年一号位经验为底，把体检结论直接落到"怎么改、找谁改"。
   - **唯一主 CTA：「开始企业诊断」**（→ diagnose.cxodex.com；阶段2 上线前先指向占位/现诊断）。次 CTA：预约沟通。
3. **三大重点区（版面主角）**：三张大卡/三段，分别讲 扭亏(精益+6σ) / 国际订单 / 组织变革——每块：一句话痛点 + 一句话打法 + 一个结果指向。**这是"体检报告会给你开的三张药方"。**
4. **典型客户滚动（突出）**：把 6 家客户做成**显眼的横向/纵向跑马灯**独立板块（金色、克制、自动滚动、悬停暂停），标题如"服务过的企业"。（复用现有 clients 数据：富耐克/艾科美/视伟/洪记/提爱思/鸿泰锆业。）
5. **单项工具区（次级，保留 5 入口）**：统一卡片列出 企业诊断/财务分析/OEE/排产数字人/面试数字人，标题如"单项深挖工具"，措辞"想单独用某个工具？"。
6. **精简的个人信任条**：把原来大段个人履历压成一行数据条（30年一号位 · GE十年 · 好孩子CEO · 16个行业），**不再重复长段自我介绍**。
7. **底部 CTA + 页脚**。

## 阶段 2：诊断向导（diagnose.cxodex.com，HK 新 app）

「开始企业诊断」进入,4 步递进（登门槛：先零成本小赢，再深入）：
- **Step 1 · OEE/产能(极简)**：设备效率 / 时间利用率 / 性能损失。凭记忆 2 分钟填，出戏剧性钩子（"OEE 52%，标杆 80%+"）。复用 oee-web 计算口径的精简版。
- **Step 2 · 财务穿透(极简)**：成本结构 / 单订单利润 / 现金流压力（利润·成本·订单几项）。复用 finance 计算精简版。
- **Step 3 · 瓶颈识别(TOC)**：定位唯一约束工序（通常=激光/磨削）。
- **Step 4 · 系统诊断(精益+6σ)**：归类 浪费型(Lean) / 波动型(6σ) / 结构型(组织·产品)。

架构：单 index.html + server.js（静态 + /api/diagnose AI）+ PM2 + nginx + certbot，DNS 新增 A `diagnose`→43.160.214.195（用户在 GoDaddy 加）。AI 走香港 glm-5.2/qwen（快、不超时）。

## 阶段 2 引擎：CXO Engine（唯一系统，判断层）
向导所有输入汇入**一个引擎**，内部三模块（各司一职、可独立测试）：
- **Module A · Flow Engine（生产流）**：OEE 计算 + 瓶颈分析（TOC 找唯一约束工序）→ 产出 Efficiency/Quality 信号、产能瓶颈根因。对应 Step1+Step3。
- **Module B · Finance Engine（利润流）**：利润分析 + 成本拆解（单订单利润、现金流压力）→ 产出 Financial 信号、利润损失根因。对应 Step2。
- **Module C · Strategy Engine（战略流）**：产品结构 + 国际订单策略 + 组织问题 → 产出 Execution 信号、结构型/组织型根因，并驱动"根因→服务"路由。对应 Step4 的结构型判断。
- 三模块 → 汇总成《CXO Diagnostic Report》：四项健康分 + 三大根因唯一化 + 90 天路径 + 三服务路由。

### 阶段 2 输出：《CXO Diagnostic Report 企业体检报告》
1. **企业健康指数 0–100**：Efficiency / Quality / Financial / Execution 四项分（规则计算，输入来自 Step1-2）。
2. **三大根因（必须唯一化）**：定位到具体环节，如"产能瓶颈在数控磨削段""利润损失在某 3 类订单""组织问题在计划决策层"（规则定位 + AI 归纳成一句话）。
3. **90 天改善路径**：产能提升 / 成本下降 / 订单结构优化（AI 生成行动计划）。
- **根因→服务路由**：
  - 运营/质量信号（OEE 低、停机集中在换型/故障）→ **①精益+6σ 扭亏**
  - 财务信号但产能 OK（毛利率低、产能利用尚可）→ **②产品战略与国际订单**
  - 组织信号（人均产值、离职率、跨部门协同）→ **③组织人事变革**
